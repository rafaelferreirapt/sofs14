Detailed Description

Display the contents of a block/cluster from a storage device in different formats.

(interface file)

The formats are:

  • block/cluster contents as hexadecimal data
  • block/cluster contents as ascii/hexadecimal data
  • block/cluster contents both as hexadecimal and ascii data
  • block contents as superblock data
  • block contents as inode data
  • single inode data
  • cluster contents as a byte stream
  • cluster contents as a sub-array of data cluster references
  • cluster contents as a sub-array of directory entry data.

void printAscii ( void *  buf,

Display the block/cluster contents as ascii/hexadecimal data.

The contents is displayed in rows of 32 characters each. Each row is labeled by the address of the first byte, displayed in decimal.

Ascii is selected by default, hexadecimal is only used when the byte value, seen as a character, has no graphic representation.

Parameters
bufpointer to a buffer with block/cluster contents
isClustertype of unit to display
  • true, the unit is a cluster
  • false, the unit is a block
void printBlkInode ( void *  buf)

Display the block contents as inode data.

The contents is displayed inode by inode and, within each inode, field by field, not literally, but in an operational way. Both decimal and ascii representations are used as required.

Parameters
bufpointer to a buffer with block contents
void printCltByteStr ( void *  buf)

Display the cluster content as a byte stream.

The header is displayed first in a single row. The body is displayed next as a byte stream in rows of 16 characters each. Each row is labeled by the address of the first byte, displayed in hexadecimal. Both decimal and ascii representations are used as required.

Parameters
bufpointer to a buffer with the cluster contents
void printCltDirEnt ( void *  buf)

Display the cluster content as a sub-array of directory entries.

The header is displayed first in a single row. The body is displayed next a directory entry per row. Only the characters of the name field that have a graphical are represented by themselves, the remaining one are replaced by the space caracter. Both decimal and ascii representations are used as required.

Parameters
bufpointer to a buffer with the cluster contents
void printCltRef ( void *  buf)

Display the cluster content as a sub-array of data cluster references.

The header is displayed first in a single row. The body is displayed next in rows of 8 references each. Each row is labeled by the address of the first reference. Both the address and the references are displayed in decimal.

Parameters
bufpointer to a buffer with the cluster contents
